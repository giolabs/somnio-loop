Run Report Examples

Every run writes two artifacts to .loop/: a human-readable run-report.md entry appended to run-log.md, and a machine-readable trace.json in .loop/traces/. All content below is illustrative.

## 2026-07-01T14:32:11Z — MYP-204
# ^ ISO-8601 timestamp + ticket ID. Appended by spec-writer at end of run.
 
- **Status:** completed
# completed | partial | aborted
 
- **Readiness level:** L2
# L1 = report-only, L2 = assisted, L3 = full autonomy. Set by triage.
 
- **Archetype phases:** generate-spec → self-healing
# Triage decomposed MYP-204 into two phases: spec first, then code.
 
- **Duration:** 4m 18s
- **Tokens:** 187,430 total — Haiku: 42,100, Sonnet: 131,200, Opus: 14,130
# Haiku = verifier audit. Sonnet = workers + triage + runtime. Opus = spec-writer only.
 
- **Cost estimate:** $0.44
- **Artifacts produced:**
  - .planning/phases/MYP-204-spec.md   # 20-section spec
  - src/services/EmailVerificationService.ts
  - src/services/__tests__/EmailVerificationService.test.ts
 
- **Verifier blocking findings:** 0
# Non-zero here means the verifier flagged issues. Self-healing may have fixed them.
 
- **Self-healing retries:** 1
# One file needed a corrective dispatch (type error fixed on retry 1 of 3).
 
- **User overrides:** none
# Records any gate decisions the user manually overrode during the run.
 
- **Trace:** .loop/traces/20260701T143211-myp204.json
- **Plan archive:** .loop/plans/20260701T143211-myp204.yml
