## Descubrimiento en tiempo de ejecuciĂłn

1. Los PRs siempre se abren como draft. Los usuarios marcan "ready for review" manualmente.
2. `gh pr merge`, `git push origin main` y cualquier llamada equivalente a merge nunca se hacen.
3. El status del ticket nunca se establece en "Done" automĂˇticamente â€” "In Review" es el mĂˇximo.
4. La limpieza de branches nunca se realiza de forma autĂłnoma.
